# Telegram VPN bot

Асинхронный Telegram-бот для автоматизации продажи VPN-подписок на базе Amnezia / Xray. Поддерживает оплату по СБП и USDT, ручную модерацию чеков администраторами, систему тикетов поддержки и фоновый планировщик истечения подписок.

### Шаг 2 — Настройка `.env`

Скопируйте шаблон и откройте в редакторе:

```bash
cp .env.example .env
nano .env   # или любой другой редактор
```

Обязательные поля:

```env
BOT_TOKEN=<токен от @BotFather>
ADMIN_IDS=<ваш Telegram ID>
```

Полное описание всех переменных — в `.env.example`.

### Шаг 3 — Наполнение пула VPN-ключей

Создайте файл `data/clients_{code}.json` с массивом слотов:

```json
[
    {
        "id": 1,
        "vpn_key": "vless://...",
        "telegram_id": null,
        "status": "inactive",
        "expires_at": null,
        "comment": ""
    }
]
```

> При первом запуске папки `data/` и `logs/` создаются автоматически.
> Если `clients_{code}.json` отсутствует — создаётся пустой массив `[]`.

### Шаг 4 — Запуск

```bash
python main.py
```

## 🔑 4. Полный справочник переменных окружения

| Переменная     | Обязательная | Описание                                 | Пример                |
| -------------- | ------------ | ---------------------------------------- | --------------------- |
| `BOT_TOKEN`    | ✅            | Токен бота от @BotFather                 | `123456:ABC...`       |
| `ADMIN_IDS`    | ✅            | ID администраторов через запятую         | `987654321,112233445` |
| `BOT_USERNAME` | -            | Юзернейм бота без @                      | `Telegram_vpn_bot`    |
| `PRICE_RUB`    | —            | Цена подписки в рублях (по умолч. `100`) | `150`                 |
| `PRICE_USDT`   | —            | Цена подписки в USDT (по умолч. `1.5`)   | `2.0`                 |
| `SBP_NUMBER`   | —            | Номер телефона СБП                       | `+79876543210`        |
| `SBP_BANK`     | —            | Название банка (отображается в боте)     | `А-Банк`              |
| `SBP_NAME`     | —            | Имя получателя (отображается в боте)     | `Иван Т.`             |
| `WALLET_TRC20` | —            | Адрес USDT TRC-20                        | `TWxA7g...`           |
| `WALLET_ERC20` | —            | Адрес USDT ERC-20                        | `0x386k...`           |
| `WALLET_BEP20` | —            | Адрес USDT BEP-20                        | `0x386k...`           |
| `WALLET_TON`   | —            | Адрес TON                                | `UABF...`             |

**Поведение незаполненных полей:**
- `PRICE_RUB` / `PRICE_USDT` — используются значения по умолчанию.
- `SBP_BANK` / `SBP_NAME` — строки пропускаются в тексте реквизитов.
- `WALLET_*` — если кошелёк не задан (пустая строка или строка отсутствует), соответствующая сеть **не появляется** в меню выбора сети.
- Если не задан ни один `WALLET_*` — кнопка «USDT» в меню оплаты **скрывается** полностью.

## 🔄 5. Системные алгоритмы

### Биллинг (Payment Flow)

```
Пользователь → Выбор метода (СБП / USDT) → Скриншот чека
    → Слот резервируется в статусе "check" → Уведомление администраторам
    → Администратор: [Одобрить] или [Отклонить]
        ✅ Одобрить → статус "active", VPN-ключ отправляется пользователю,
                      запись в history.json (защита от повторного триала)
        ❌ Отклонить → статус "inactive" (слот освобождается), уведомление пользователю
```

> **Важно:** VPN-ключ отправляется пользователю **только после одобрения** чека администратором, а не при отправке скриншота.

### Планировщик подписок (ежедневно в 00:01)

| Условие | Действие |
|---|---|
| До истечения осталось 3 дня | Мягкое напоминание пользователю |
| До истечения остался 1 день | Срочное предупреждение |
| `expires_at` наступила | Статус → `expired`, уведомление пользователю и всем администраторам |

Каждый `bot.send_message` обёрнут в `_safe_send()` — `TelegramForbiddenError` (пользователь заблокировал бота) логируется и не прерывает обработку остальных слотов.

## 🗄️ 6. Схема данных

### `clients_{code.json` — пул VPN-слотов

```json
{
    "id": 1,
    "vpn_key": "vless://...",
    "telegram_id": 123456789,
    "status": "active",
    "expires_at": "2025-08-01",
    "comment": "@username (Иван)"
}
```

| Поле | Тип | Описание |
|---|---|---|
| `id` | int | Уникальный номер слота |
| `vpn_key` | str | Строка конфигурации Amnezia/VLESS |
| `telegram_id` | int \| null | ID привязанного пользователя |
| `status` | str | `inactive` / `check` / `active` / `expired` / `banned` |
| `expires_at` | str \| null | Дата истечения `YYYY-MM-DD` |
| `comment` | str | `@username (Имя)` — для удобства администратора |

### `history.json` — лог активаций

```json
{"telegram_id": 123456789, "activated_at": "2025-07-01T12:00:00 [TRIAL]"}
```

Используется для защиты от повторного получения пробного периода. Запись добавляется атомарно — race condition при двойном нажатии исключён.

### `support.json` — тикеты поддержки

```json
{
    "ticket_id": 1,
    "telegram_id": 123456789,
    "username": "ivan",
    "full_name": "Иван Иванов",
    "status": "open",
    "created_at": "2025-07-01T12:00:00",
    "messages": [
        {"sender": "user", "text": "Не работает VPN", "timestamp": "..."},
        {"sender": "admin", "text": "Проверьте ключ", "timestamp": "..."}
    ]
}
```

### `countries.json`

```json
[
  {"code": "nl", "flag": "🇳🇱", "label": "Нидерланды"},
  {"code": "de", "flag": "🇩🇪", "label": "Германия"}
]
```

Определяет доступные локации.

## 🔄 7. Логика основных процессов

### 🎁 Пробный период (Trial Flow)

1. Проверка через history.json: если пользователь уже получал триал или покупал подписку — отказ.
    
2. Поиск в countries.json первой страны со свободным (inactive) слотом.
    
3. Присвоение слоту статуса trial, запись expires_at (сегодня + TRIAL_DAYS).
    
4. **Реферальный бонус:** Если пользователя пригласили, его рефереру мгновенно начисляется **+3 дня** к его активной подписке.
    

### 💳 Оплата и Модерация (Payment Flow)

1. Пользователь выбирает страну -> Метод оплаты -> Присылает скриншот.
    
2. Слот в выбранной стране переходит в статус check (резервируется).
    
3. Админы получают фото чека и кнопки [Одобрить | Отклонить].
    
4. **При одобрении:** статус active, срок +30 дней, отправка ключа юзеру.
    
5. **Реферальный бонус:** Рефереру начисляется **+7 дней** (только за первую оплату друга).
    

### 🕒 Планировщик (Scheduler Flow)

Работает в фоне (запуск в 00:01 ежедневно):

- **Reminders:** За 3 и 1 день до конца подписки бот присылает уведомление и кнопку «Продлить».
    
- **Expiration:** В день expires_at статус меняется на expired, доступ блокируется (уведомление юзеру и админу).

## 🚨 8. Гайд по расширению

### Добавление нового хендлера

1. Создайте файл `src/handlers/referral.py` с локальным `router = Router()`.
2. Зарегистрируйте его в `src/handlers/__init__.py`:
   ```python
   from .referral import router as referral_router
   main_router.include_routers(..., referral_router)
   ```
3. В `main.py` ничего менять не нужно.

### Добавление новой сети USDT

Достаточно добавить переменную в `.env`:
```env
WALLET_SOL=<адрес кошелька Solana>
```
И добавить обработку в `config.py` (`WALLETS["sol"] = ...`) и метку в `payment.py` (`_LABELS["sol"] = "SOL"`).

### Смена метода хранения данных

Все операции с данными изолированы в `src/database.py`. Для перехода на `aiosqlite` или PostgreSQL достаточно переписать только этот модуль, не трогая хендлеры.
